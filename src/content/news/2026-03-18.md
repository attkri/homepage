## Breaking News

### Topic-Cluster: Microsoft baut Copilot um, stoppt Zwangsinstallation und schränkt kostenlosen Zugang ein

Microsoft hat in einer Woche gleich drei Copilot-Entscheidungen getroffen, die zusammen ein klares Signal senden: Mustafa Suleyman gibt die direkte Copilot-Verantwortung ab, Jacob Andreou übernimmt als EVP Copilot. Die automatische Copilot-App-Installation auf Windows-Systemen mit M365-Desktop-Apps wurde temporär gestoppt. Und ab 15. April 2026 wird Copilot in Word, Excel, PowerPoint und OneNote nur noch für zahlende M365-Copilot-Lizenzen verfügbar sein – Nutzer ohne Lizenz bekommen nur noch "Copilot Chat (Basic)". Das ist kein Rückzug aus KI, sondern eine Neuausrichtung: weg von Zwangsintegration, hin zu zahlenden Enterprise-Kunden.

Community-Reaktion: In Admin-Foren und Tech-Blogs überwiegt Erleichterung über das Ende der Zwangsinstallation, aber auch Skepsis, ob die Umbenennung in "Basic" vs. "Premium" wirklich eine Verbesserung ist oder nur eine neue Vermarktungsstrategie.

## IT-Security

### [Cyberangriffe 2026: Identitäten statt Firewalls im Fokus](https://borncity.com/news/cyberangriffe-2026-identitaeten-statt-firewalls-im-fokus/)

Der Angriff auf Stryker Corporation durch die Iran-nahe Gruppe Handala zeigt das neue Muster: Kompromittiertes Admin-Konto in Microsoft Intune, Fernlösch-Befehle für 5.000 Geräte, 5.000 Mitarbeiter nach Hause geschickt – ohne klassische Ransomware. Parallel zerschlug eine internationale Polizeiaktion die Phishing-as-a-Service-Plattform Tycoon 2FA, die für 62 Prozent aller von Microsoft blockierten Phishing-Versuche verantwortlich war. Die durchschnittliche Breakout-Zeit nach erstem Zugriff liegt jetzt unter 60 Minuten. Perimeter-Verteidigung ist damit endgültig überholt.

## Tools & Dev

### [Azure DevOps Remote MCP Server in Public Preview](https://devblogs.microsoft.com/devops/azure-devops-remote-mcp-server-public-preview/)

Microsoft hat den Azure DevOps Remote MCP Server in die Public Preview gebracht. Der gehostete MCP-Server nutzt streamable HTTP-Transport und Microsoft Entra für Authentifizierung – keine lokale Installation mehr nötig. Unterstützt werden aktuell Visual Studio und VS Code mit GitHub Copilot. Claude Desktop, Claude Code und ChatGPT folgen, sobald die OAuth-Client-ID-Registrierung in Entra verfügbar ist. Für Teams, die Azure DevOps und KI-Agenten kombinieren wollen, ist das der direkteste Weg.
